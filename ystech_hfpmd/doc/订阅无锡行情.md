# 业务需求
行情订阅程序
* 输入：读取配置文件里的账户信息，
* 处理：每天定时启动, 订阅交易所下所有行情。
* 输出：每天定时关闭，压缩归档行情文件。

# 非功能需求
* 使用git管理代码版本。
* 充分复用misc公共代码库。
* 在linux下运行的c++程序。
* 要考虑断线重连。如果能够提示订阅失败就更好了。
* 要有日志记录程序运行中的事件。
* 使用ansible部署行情订阅程序。
